# 剪辑稿与项目文件格式

## 目录

1. `project.md`
2. 主题地图
3. 逐字剪辑稿
4. 原文核验报告
5. 剪辑单

## 1. `project.md`

```markdown
# 项目状态

- 源素材：/absolute/path/interview.mp4
- 源逐字稿：edit/transcripts/interview.txt
- 目标平台：抖音
- 最终目标：180 秒 / 约 840–900 内容单位
- 候选稿预算：约 5 分钟 / 不超过 1500 内容单位
- 当前门槛：1 主题待确认
- 已选主题：待定
- 待复听：无

## 已批准决策

- YYYY-MM-DD：用户选择主题 A。

## 下一步

- 用户确认灵魂四问后，生成候选剪辑稿。
```

## 2. 主题地图

每个主题使用同一结构：

```markdown
## T01 主题名

- 一句话主张：
- 核心冲突：
- 对谁说：
- 想说什么：
- 希望对谁产生什么影响：
- 对立面：
- 原文证据：时间码 / 原句摘要（此处摘要不进入成片口播）
- 情绪峰值：
- 潜在钩子：
- 跑偏风险：
- 编辑建议：推荐 / 备选 / 不推荐
```

## 3. 逐字剪辑稿

只有 HTML 标记之间的文字属于成片口播；标题、解释和备注不计入字数。

```markdown
# 主题：为什么他能改变这个行业？

## C001 | HOOK | 00:12:03.200–00:12:12.800

<!-- VERBATIM:C001 -->
我第一次见到他的时候，我就觉得这个人跟别人不一样。
<!-- /VERBATIM -->

- 作用：制造人物悬念
- 画面：说话人前倾，保留句尾停顿
- 衔接：悬念 → 具体事件

## C002 | SETUP | 00:05:10.000–00:05:24.500

<!-- VERBATIM:C002 -->
那一年我们整个团队都快撑不下去了，他做了一个所有人都反对的决定。
<!-- /VERBATIM -->

- 作用：交代处境和冲突
```

规则：

- 每个 `VERBATIM` 块只放一个源位置连续说出的原句。
- 允许调整块顺序，不允许在块内改字。
- 想从同一长句中间删字时，拆成两个块，并分别给出可回剪的时间码；确认剪点不会落在词中。
- 无可靠时间码时写 `NO-TIMECODE`，不能猜。
- 需要复听时在标题末尾写 `[待复听]`。

## 4. 原文核验报告

记录脚本命令、结果、缺失块、重复块、内容单位和估算时长。若有缺失块，剪辑稿状态必须是“未通过”，不得只在备注里淡化。

## 5. 剪辑单

| 顺序 | 片段 ID | 源文件 | 入点 | 出点 | 逐字原句 | 结构角色 | 情绪/画面 | 衔接 |
|---:|---|---|---|---|---|---|---|---|
| 1 | C001 | interview.mp4 | 00:12:03.200 | 00:12:12.800 | 我第一次…… | 钩子 | 前倾；保留停顿 | 接 C002 |

不要只写“见原稿”。剪辑师必须能凭这一行找到并判断片段。
